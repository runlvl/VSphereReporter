Information zur Version v29.0

Während der Entwicklung der Version v29.0 wurden verschiedene Debugging-Versionen erstellt, um spezifische Probleme zu beheben. Wählen Sie bitte die Version entsprechend Ihres Anwendungsfalls aus.

NEU: VMware vSphere Reporter v29.0 (Build 18)
MAXIMALE DIAGNOSE - Zeigt rohe API-Daten an

Diese grundlegende Diagnoseversion ist entwickelt, um technische Probleme in der Anwendung zu umgehen. Sie bietet:

  • Direkter Zugriff auf unverarbeitete API-Daten ohne Fehleranfällige Verarbeitung
  • Geradliniger, vereinfachter Login-Prozess
  • Detaillierte Fehlerprotokolle für jeden API-Aufruf
  • Umgeht bekannte Probleme in Build 16/17
Empfohlen für Fehlerfälle! Diese Version sollte auch dann funktionieren, wenn die anderen Versionen komplett fehlschlagen. Ideal für Grundlagendiagnose.
  • Veröffentlicht am 06.05.2025
  • Version v29.0 Build 18

VMware vSphere Reporter v29.0 (Build 17)
ALTERNATIVE Implementierung für VMDK-Erkennung

Diese Version verwendet einen völlig anderen Ansatz zur Erkennung von VMDK-Dateien. Sie sammelt zuerst alle registrierten VMDKs direkt von VM-Konfigurationen und durchsucht dann alle Datastores separat mit einer SubFolder-Suche. Dadurch sollten mehr VMDKs gefunden werden.

Empfohlen für Problemlösung! Diese Version verwendet eine andere API (SearchDatastoreSubFolders_Task) für eine umfassendere Suche in allen Unterverzeichnissen.
  • Veröffentlicht am 06.05.2025
  • Version v29.0 Build 17
Herunterladen
VMware vSphere Reporter v29.0 (Build 16)
EXTREME Debug-Version für VMDK-Probleme

Diese spezielle Version enthält ein sehr ausführliches Logging auf DEBUG-Level und speichert Stack-Traces für alle auftretenden Fehler. Verwenden Sie diese Version, wenn mit der Debug-Version v15 keine VMDKs angezeigt werden, um die Fehlerursache zu identifizieren.

Nur für Troubleshooting! Diese Version erzeugt sehr große Log-Dateien und sollte nur zur Fehlerdiagnose verwendet werden.
  • Veröffentlicht am 06.05.2025
  • Version v29.0 Build 16
